Mastra AI engine

The system gets smarter each article

Mastra's reinforcement learning loop turns every correction you make into a permanent improvement for your brand.

1
AI generates a draft
Using your template, brand voice, and research brief. The prompt already includes learnings from all previous articles for this brand.
2
You make corrections
You adjust tone, restructure a section, fix a claim, or change a heading in the inline editor. Each change is recorded as a correction signal.
3
Mastra stores the pattern
The difference between what was generated and what you preferred is stored in your brand's correction memory. Mastra uses this to improve future prompts.
4
Next article starts better
The same mistakes don't repeat. The generation prompts for Stream A and Stream B both receive your brand-specific correction history before producing output.
5
Quality compounds over time
After 5–10 articles, the system has a rich model of your exact preferences — tone, structure, fact standards, and formatting — without any manual configuration.
6
Fewer edits at each gate
As corrections decrease, you spend less time at Gate 2. The approval flow gets faster. The loop reinforces itself — better output → fewer corrections → even better output.

GitHub connection

Two ways to connect your repository

ContentPilot needs write access to commit MDX files. Choose the method that fits your team's setup.

Recommended
🔌
GitHub App (MCP)
Install the ContentPilot GitHub App from the GitHub Marketplace. It requests only the specific repository permissions needed — nothing else. You authorize access to one repo, and can revoke it instantly from your GitHub settings at any time.
  • ✓ No token to copy, manage, or rotate — ever expires
  • ✓ Permission-scoped to one repo only
  • ✓ Works for the whole team — not tied to one person's account
  • ✓ Revoke in one click from GitHub → Settings → Apps
  • ✓ Audit log of all commits made by the app
Setup: Brand Settings → Integrations → Connect GitHub → Authorize App → Select repository. Done.
🔑
Personal / Org access token
Generate a fine-grained GitHub personal access token and paste it into ContentPilot's brand settings. Works without installing any app — useful for private or self-hosted GitHub instances.
  • ✓ No app installation required
  • ✓ Works with GitHub Enterprise / self-hosted
  • ✗ Token is tied to one user account — access breaks if they leave
  • ✗ Tokens expire and must be manually rotated
  • ✗ Classic tokens grant broader access than needed — use fine-grained
Token setup: GitHub → Settings → Developer settings → Fine-grained tokens → New token. Set repository access to your blog repo only. Enable Contents: read & write. Copy and paste into ContentPilot brand settings.

Error handling

What happens when something goes wrong

Every failure mode has a defined behaviour. Content is never silently lost.

Level 1
AI generates bad JSON
If the frontmatter JSON fails schema validation, Mastra auto-retries the step up to 2 times. On third failure, the BullMQ job retries with exponential backoff (max 3 attempts). If all fail, status moves to FAILED and you're notified.
Level 2
GitHub API error
401 Unauthorized → hard fail with message "GitHub token invalid — update in Brand Settings." 404 Not Found → hard fail "Repository not found — check owner/repo name." 422 Conflict → fetches latest SHA and retries once automatically.
Level 3
Image CDN upload fails
Non-blocking. The pipeline continues without that image. The article still commits to GitHub. Failed image details are logged in BlogQueue.metadata.imageGenerationError so you can investigate separately.
Level 4
Complete pipeline failure
If all retries are exhausted, the article draft is preserved in the database — content is never lost. Status moves to FAILED and the dashboard shows an "Attention Required" warning with the exact error reason.
